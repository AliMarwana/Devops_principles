# Devops_principles
1. DevOps is a culture that aims to improve development and operations in the IT context.
   Here are the core principles of DevOps:
   _Automation: here we automate tasks which used to be done manually in traditional IT. It makes bugs correction faster and saves time.\
   _Collaboration: it allows collaboration between developers and operation teams. Therefore the developer does not have to wait for the end of the project to see the operation team work on the production. \
   _Continuous improvement: it focuses on the improvement of the software. As soon as bugs are detected, corrections are made.\
2. Here are the devOps lifecycle stages:\
   Planning: here we plan requirements. We create user stories and prioritize tasks.\
   Development: here we code the project. It is actually the implementation of the planning.\
   Building: here we compile the code. We can also run automatic builds.\
   Testing: here we run the automatic tests. We check security and performance validation.\
   Release: here we package and store the approved artifact.\
   Deployment: here we install the software in an environment in production. It may be accessible from the link of a website for example.\
   Operations: here we run and maintain the live system.\
   Monitoring: here we observe the logs and performance.\
